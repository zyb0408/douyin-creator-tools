# Douyin 作品获取业务流程

## 1. 参数解析
- 通过 `npm run works -- --limit N` 启动脚本
- CLI 参数 `--limit N` 被 `list-douyin-works.mjs` 中的 `parseArgs` 正确解析为 `args.limit`
- 参数传递必须使用 `--` 分隔符，确保参数传入 Node.js 脚本而非 npm

## 2. 工作流启动
- `list-douyin-works.mjs` 加载 `works-panel.mjs` 中的 `listWorks` 函数
- 传入包含 `limit`、`profileDir`、`outputPath` 等的参数对象

## 3. 作品获取（核心逻辑）
- 调用 `fetchAllWorks(limit)`，启动无限滚动抓取流程
- **步骤 1**：滚动至页面顶部，确保获取最新作品
- **步骤 2**：首次读取当前页面所有可见作品，统计数量
- **步骤 3**：判断当前作品数 ≥ limit？
  - ✅ 是：立即返回前 `limit` 条作品，**停止滚动**
  - ❌ 否：执行滚动加载
- **步骤 4**：执行一次向下滚动，等待内容加载
- **步骤 5**：重新读取作品列表，累加总数
- **步骤 6**：重复步骤 3–5，直到：
  - 达到 `limit` 数量，或
  - 无新内容加载（已到底部）

## 4. 数据输出
- 将最终获取的作品列表（最多 `limit` 条）序列化为 JSON
- 输出路径：`args.outputPath`（默认：`comments-output/list-works.json`）
- 文件写入完成后，脚本正常退出

## 5. 异常处理
- 所有网络/页面异常由 Playwright 自动重试机制处理
- 未捕获异常将打印堆栈并退出，退出码为 1
- 无作品时输出空数组，不报错

## 6. 性能与稳定性
- 每次滚动后等待 1.5s，确保内容渲染完成
- 首次加载后等待 2s，确保顶部内容刷新
- 仅读取可见 DOM，不加载全页，避免内存溢出
- 严格按 `limit` 截断，不缓存或预加载多余数据

## 7. 验证规则
- 所有输出文件必须为合法 JSON
- 输出作品数 ≤ `limit`
- 作品顺序为从最新到最旧（顶部优先）
- 无重复作品（依赖抖音前端去重机制）

## 8. 扩展建议
- 可增加 `--since-id` 参数，支持断点续抓
- 可增加 `--output-format=jsonl` 支持流式输出
- 可集成日志系统，支持 `--debug` 详细模式
